# 对 GitHub PR 做 CR

用于根据用户提供的 GitHub PR 地址做 CR。

## 流程

1. 确认用户提供了 GitHub PR 地址。

2. 获取 PR 的目标分支、源分支和本次 PR 的完整改动范围。

3. 只看这个 PR 引入的改动，然后做 CR。

4. 输出固定的 Markdown 问题列表。

## CR问题列表输出格式

输出必须是 Markdown，并使用固定列表结构：

```md
## CR 问题列表

1. **级别**：严重 / 中等 / 轻微
   **位置**：文件路径:行号
   **问题**：说明具体问题。
   **影响**：说明可能造成的影响。
   **复现路径**：说明如何触发或验证这个问题。
   **建议**：说明建议如何修改。
   **一句话总结**：用通俗、容易理解的方式解释这个问题。
```

如果没有发现问题，输出：

```md
## CR 问题列表

未发现明显问题。
```

## 注意

- 只 review 这个 PR 引入的改动，不要混入 PR 范围外的历史改动。
- 需要同时关注代码 diff、PR 描述、相关提交和测试变化。
- 避免：自动部署、push、merge、rebase、reset。
- 避免给出非本批次修改引入的问题，避免给出历史问题。
- 确保影响已有功能的问题都被识别出来。
