# 对当前修改做 CR

用于 review 当前工作区里尚未提交的修改。

## 流程

1. 只看当前工作区里尚未提交的修改，然后做 CR。

2. 输出固定的 Markdown 问题列表。

## CR问题列表输出格式

输出必须是 Markdown。问题列表部分使用固定列表结构，列表之后可以追加其他自由反馈：

```md
## CR 问题列表

1. **级别**：严重 / 中等 / 轻微
   **位置**：文件路径:行号
   **问题**：说明具体问题。
   **影响**：说明可能造成的影响。
   **复现路径**：说明如何触发或验证这个问题。
   **建议**：说明建议如何修改。
   **一句话总结**：用通俗、容易理解的方式解释这个问题。
```

如果没有发现问题，输出：

```md
## CR 问题列表

未发现明显问题。
```

问题列表之外可以继续输出简短评论、已修复项、验证说明等自由内容，不受固定格式限制。

## 注意

- 只 review 当前未提交修改，不要混入已经提交的历史改动。
- 避免：自动部署、push、merge、rebase、reset。
- 避免给出非本批次修改引入的问题，避免给出历史问题。
- 确保影响已有功能的问题都被识别出来。
