# 对指定 Commit 之后做 CR

用于 review 当前分支上从用户指定的某个 commit 开始，包含这个 commit，一直到当前最新提交的改动。

## 流程

1. 确认用户提供了起始 commit。

2. 只看从这个 commit 开始，包含这个 commit，到当前分支最新提交之间的改动，然后做 CR。

3. 输出固定的 Markdown 问题列表。

## CR问题列表输出格式

输出必须是 Markdown，并使用固定列表结构：

```md
## CR 问题列表

1. **级别**：严重 / 中等 / 轻微
   **位置**：文件路径:行号
   **问题**：说明具体问题。
   **影响**：说明可能造成的影响。
   **复现路径**：说明如何触发或验证这个问题。
   **建议**：说明建议如何修改。
   **一句话总结**：用通俗、容易理解的方式解释这个问题。
```

如果没有发现问题，输出：

```md
## CR 问题列表

未发现明显问题。
```

## 注意

- 只 review 指定 commit 到当前最新提交之间的改动，不要混入范围外的历史改动。
- 指定 commit 本身也包含在 review 范围内。
- 避免：自动部署、push、merge、rebase、reset。
- 避免给出非本批次修改引入的问题，避免给出历史问题。
- 确保影响已有功能的问题都被识别出来。
